# Contributing

Keep changes small and grounded in the skill contract.

## Rules

- Do not weaken safety tiers.
- Do not make architecture edits auto-safe.
- Keep `SKILL.md` concise; put detailed material in `skills/improve-codebase-health/references/`.
- Update README/docs when install or invocation changes.
- Run `npm run check` before commit.

## Useful Commands

```bash
npm run check
```
